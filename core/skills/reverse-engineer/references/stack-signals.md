# Stack Detection Signals

Check these files/patterns in order to identify the tech stack.

## fastapi-vue (Python + FastAPI + Vue3)

- `backend/requirements.txt` or `backend/pyproject.toml` contains `fastapi`
- `backend/app/main.py` imports `FastAPI` and creates `app = FastAPI()`
- `backend/app/models/` contains SQLAlchemy `Mapped[]` type annotations
- `backend/alembic.ini` or `backend/alembic/` directory exists
- `frontend/package.json` contains `"vue"` and `"vite"`

## node-vue (TypeScript + Fastify + Vue3)

- `backend/package.json` contains `"fastify"`
- `backend/src/db/schema/` contains Drizzle table definitions (`pgTable(...)`)
- `backend/drizzle.config.ts` exists
- `frontend/package.json` contains `"vue"` and `"vite"`

## Legacy stacks (detection only — not generated by this marketplace)

### vaadin-jooq
- `pom.xml` with `com.vaadin` dependency
- Java source files with `@UseCase` or `@Route` annotations
- `src/main/resources/db/migration/` Flyway SQL files

### spring-boot
- `pom.xml` or `build.gradle` with `spring-boot-starter`
- `@SpringBootApplication` annotation in a Java class
- `application.yml` or `application.properties`

### django
- `manage.py` in project root
- `settings.py` with `INSTALLED_APPS`
- `models.py` files with `django.db.models.Model` subclasses

### rails
- `Gemfile` containing `gem 'rails'`
- `config/routes.rb`
- `app/models/` with ActiveRecord classes

### go
- `go.mod` in project root
- HTTP handler functions (`func handler(w http.ResponseWriter, r *http.Request)`)
- `main.go`

### dotnet
- `*.csproj` file
- `Program.cs` with `WebApplication.CreateBuilder`
- `appsettings.json`
